# 3D Model Viewer

Приложение для просмотра и редактирования 3D моделей в формате OBJ.

## Возможности

- Загрузка и сохранение OBJ файлов
- Поддержка нескольких моделей одновременно
- Выбор и управление активной моделью
- Удаление вершин и полигонов
- Светлая и темная темы интерфейса
- Обработка ошибок с понятными сообщениями


## Экспорт моделей из Blender

Для корректной работы текстур и освещения:

1. При экспорте OBJ из Blender убедитесь, что включены опции:
   - **Include Normals** (Write Normals)
   - **Include UVs**
   - **Write Materials** (создаст .mtl файл)
   - **Triangulate Faces** (рекомендуется)
2. Все текстуры должны быть в той же папке, что и OBJ/MTL, либо путь к ним должен быть относительным.
3. В MTL-файле должна быть строка `map_Kd имя_текстуры.png` для диффузной текстуры.
4. Загружайте OBJ-файл через меню программы — текстура и освещение применятся автоматически.

## Требования

- Java 21 или выше
- JDK для сборки проекта

## Сборка и запуск

### Windows

1. **Сборка проекта:**
   ```
   build.bat
   ```

2. **Запуск приложения:**
   ```
   run.bat
   ```
   Или напрямую:
   ```
   java -jar dist/ModelViewer.jar
   ```

### Linux/macOS

1. **Сборка проекта:**
   ```bash
   chmod +x build.sh
   ./build.sh
   ```

2. **Запуск приложения:**
   ```bash
   chmod +x run.sh
   ./run.sh
   ```
   Или напрямую:
   ```bash
   java -jar dist/ModelViewer.jar
   ```

## Использование

1. **Загрузка модели:**
   - Меню `File` → `Open Model...` (Ctrl+O)
   - Выберите OBJ файл

2. **Сохранение модели:**
   - Выберите активную модель из списка
   - Меню `File` → `Save Model...` (Ctrl+S)

3. **Удаление вершин/полигонов:**
   - Выберите модель из списка слева
   - В панели инструментов справа выберите индекс и нажмите соответствующую кнопку

4. **Переключение темы:**
   - Меню `View` → `Toggle Theme` (Ctrl+Shift+T)

```
│   └── view/
│       ├── ViewerWindow.java  # Главное окно
│       └── ModelRenderer.java # Компонент визуализации
├── META-INF/
│   └── MANIFEST.MF            # Манифест для JAR
├── build.bat / build.sh       # Скрипты сборки
└── run.bat / run.sh           # Скрипты запуска
```
